Archivos git: .env + .gitignore

Es común que se utilicen archivos .env para almacenar información sensible, como contraseñas, claves de API u otros datos confidenciales que deben mantenerse fuera del repositorio Git.
El archivo .gitignore es un archivo de configuración en Git que se utiliza para especificar qué archivos y directorios deben ser ignorados por el control de versiones. Cuando trabajas en un proyecto en Git, es común tener archivos y directorios que no deseas que se incluyan en el historial de versiones, como archivos generados automáticamente, archivos de configuración local o datos sensibles. El archivo .gitignore se utiliza para indicar a Git qué archivos y directorios deben ser excluidos al realizar operaciones como git add o git commit, lo que evita que se incluyan accidentalmente en el repositorio. De esta manera, se puede mantener un control más limpio y ordenado del repositorio, ya que solo se incluirán los archivos esenciales y relevantes para el proyecto.
